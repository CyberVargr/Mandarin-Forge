## The app: chinese-coach.html

A single-file HTML app (no build step, no server, double-click to open in Chrome/Edge on Windows). Everything inline — CSS, JS, data.

**Current version: v5 — "Forgebound Layer"**

### Architecture constraints — DO NOT break these

1. **Single file, no build step.** No npm, no webpack, no external files. CDN-only for: `hanzi-writer` (stroke order) and `opencc-js` (simplified→traditional). Both degrade gracefully offline.
2. **No accounts, no server.** All progress stored in `localStorage` under key `zhTrainerV4`. Migrates from V1–V3.
3. **Traditional Chinese by default.** Taiwan Mandarin (zh-TW) throughout. No erhua (兒化). Prefer Taiwan lexical variants (筆電 not 笔记本, 行銷 not 营销, 履歷 not 简历, etc.)
4. **Patch-only — never rebuild from scratch.** Apply targeted edits to the existing file. The word array order must never change (only append); SRS card indices depend on it.
5. **Audio = Web Speech API** (`speechSynthesis`), zh-TW voice preferred.
6. **No raw backticks inside template literals** — a hard-won lesson. Use string concatenation for HTML building.
7. **MCQ answers must be index-based**, never compared by textContent.

## QA rules (mandatory before any delivery)

1. `node --check` on the extracted script block.
2. All tabs render without errors on fresh state (no localStorage) AND populated state.
3. After patching: verify file size and tail agree — sandbox mounts can serve stale copies. If overwriting and the mount seems stale, mirror the patch onto a `/tmp` copy and test from there.
4. Typing must accept: toneless pinyin, tone numbers (xue2xi2), `v` for ü, TW and mainland hanzi forms.
5. Boss quiz and session engine must not throw on fresh state.
6. Radical quiz distractor guard: never show a radical visually present in the character as a wrong answer.
